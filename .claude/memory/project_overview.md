## Статус проекта

Все 5 модулей ТЗ реализованы в коде (стадия 1 завершена):

1. **База данных подрядчиков и сотрудников** — полностью реализовано (модели Organization, Employee, EmployeeWorkClass, EmployeeDocument, ApprovalRequest). CRUD + согласования + фото + документы с истечением срока
2. **Наряды-допуски** — реализовано (модели Permit, PermitApproval). Нумерация `{CAT}-{orgSeq}-{curatorSeq}-{permitSeq}`, печать, досрочное закрытие
3. **Акты о нарушениях** — реализовано (модели Violation, ViolationComplaint, ViolationTemplate). Шаблоны, жалобы подрядчиков, печать
4. **Чек-листы проверок** — реализовано (модели Checklist, ChecklistItem). Статистика, тренды, выгрузка
5. **Нормативные документы ВШЗ** — реализовано (модели RegDocument, RegDocumentSection). Иерархические разделы, только HSE загружает, все просматривают

## Что сделано до прода

- 20 моделей Prisma, 16 enum
- 37+ API маршрутов с auth + валидацией
- Полная ролевая модель (5 ролей), двухуровневая авторизация
- Фронтенд: все страницы списков, форм, деталей, печати
- Email-уведомления + in-app уведомления
- Cron jobs: истечение документов, истечение нарядов, предупреждения
- Тесты: E2E (Playwright), integration, unit
- Seed скрипт с тестовыми данными

## Что требует доработки для production

- Настройка production БД (PostgreSQL, не SQLite fallback)
- S3 конфигурация для хранения файлов
- Настройка SMTP для email
- External cron scheduler (systemd timer / cloud scheduler)
- HTTPS + reverse proxy (nginx)
- Аудит безопасности перед релизом
- Пользовательское тестирование с реальными подрядчиками
